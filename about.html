Mode Of Operation

The implementation starts by setting up the ESP8266 module by flashing the latest version of the firmware. To flash the latest firmware, download the ESP8266 flasher tool and the latest firmware from the internet which would be in the bin format and flash the ESP8266 with it. Once the ESP8266 flashing done, other components can be added to the configuration. We need a breadboard to connect the microcontroller, ultra sonic sensor and the ESP8266 using the jumper wires. The breadboard is used to interface between the various components available as illustrated in the diagrams below.

The level of the garbage is detected by the ultrasonic sensor and if it exceeds a predetermined threshold, the dustbin is determined to be full. The ESP8266 module is then configured to operate in Access Point(AP) Mode via AT Commands. In this mode, the module provides WIFI connectivity to devices like phones, laptops. A laptop already connected to the Arduino connects to the WIFI and is assigned an IP Address. This address is then typed into a web browser with port 80 open and a message is displayed to the user alerting them on whether the bin is full or not.

Connecting To The Cloud

However, the system would not be based on IOT unless it was able to communicate with the cloud. An account is then created on the ThingSpeak Cloud Platform with a channel where the sensor data will be sent from the Arduino. An HTTP request is then made to https://api.thingspeak.com with an API key for end to end encryption purposes. The sensor data is displayed as graph in the respective channel as shown.
